# 🛋️ AI Room Topology Matcher (Умный поиск интерьеров для Staging)

**AI Room Topology Matcher** — это профессиональный скрипт на базе компьютерного зрения для подбора идеальных пустых комнат-референсов. 
Проект создан специально для задач AI-стейджинга (виртуальной расстановки мебели), где критически важно 100% совпадение перспективы, угла обзора и расположения света.

## ⚠️ Какую проблему решает проект?
Стандартные нейросети для поиска похожих картинок (ResNet50) или оценки глубины (Depth Anything / MiDaS) часто ошибаются в пустых помещениях:
1. **Зеркальный эффект:** Они путают левый угол с правым.
2. **Иллюзия перспективы:** Принимают плоскую фронтальную стену с падающей тенью за уходящий вдаль коридор.
3. **Шум от текстур:** Тени на полу, паркетные узоры и отражения создают ложные линии, искажая оценку геометрии.
4. **Брак при наложении (Коллизии):** Они подбирают маску мебели, которая при наложении на новую комнату перекрывает окна или дверные проемы.

## 🧠 Наше решение: Heuristic Topology + Depth-Guided Geometry

Мы отказались от "слепого" сравнения пикселей и перевели задачу в плоскость жесткой архитектурной геометрии. 
Алгоритм использует три нейросети — **SegFormer** (сегментация), **Depth Anything V2** (карта глубины) и **MLSD** (детектор структурных линий) — только как "глаза", а все решения принимает детерминированная математика.

### Используемые модели:
| Модель | Назначение |
|---|---|
| **SegFormer B0** (nvidia/segformer-b0-finetuned-ade-512-512) | Сегментация: пол, потолок, стены, окна, двери, мебель |
| **Depth Anything V2 Small** (depth-anything/Depth-Anything-V2-Small-hf) | Карты глубины для сравнения перспективы |
| **MLSD Tiny** (Mobile Line Segment Detection) | Детекция структурных линий комнаты на depth-карте |

### Как работает алгоритм (Pipeline):

#### Этап 1: Жёсткий фильтр (Hard Filter)
Кандидаты, не прошедшие эти проверки, полностью отсеиваются:

1. **Тип комнаты (room_type):**
   По линии плинтуса алгоритм математически вычисляет координату "Дальнего угла" и классифицирует комнату как **Фронтальную** (прямая задняя стена) или **Угловую**. Несовпадение = отсев.

2. **Доминирующая стена (dominant_wall):**
   Определяется, в какую сторону уходит перспектива: **Left**, **Right** или **Symmetric**. Зеркальные комнаты мгновенно отсекаются.

#### Этап 2: Скоринг (Scoring Formula)
Для прошедших фильтр кандидатов вычисляется финальный балл:

```
final_score = floor_iou
            + (ceiling_iou × 0.3)
            + (depth_sim × 0.5)
            + (wall_sim × 0.4)
            + (vp_sim × 0.7)
            + (scale_sim × 0.4)
            + (win_pos_sim × 0.5)
            − (area_penalty × 0.5)
            − (collision × 2.0)
```

| Компонент | Вес | Описание |
|---|---|---|
| **floor_iou** | +1.0 | IoU масок пола (64×64). Совпадение формы и площади пола |
| **ceiling_iou** | +0.3 | IoU масок потолка (64×64). Помогает отличить мансардные, скошенные и обычные потолки |
| **depth_sim** | +0.5 | NCC (Normalized Cross-Correlation) depth-карт (64×64, Depth Anything V2). Совпадение перспективы |
| **wall_sim** | +0.4 | Сходство пропорций стен (wall_left_ratio vs wall_right_ratio). Камерный угол |
| **vp_sim** | +0.7 | Vanishing Point similarity — сравнение точки схода перспективы + гистограмм углов линий (Depth→MLSD) |
| **scale_sim** | +0.4 | Метрика масштаба комнаты — разброс структурных линий (x_spread, y_spread). Маленькие vs большие комнаты |
| **win_pos_sim** | +0.5 | Позиция окон — нормализованный X центра масс окон. Штрафует если окна на противоположных сторонах |
| **area_penalty** | −0.5 | Штраф за разницу в доле пола от кадра |
| **collision** | −2.0 | Штраф за перекрытие мебели кандидата с окнами/дверями запроса |

#### Этап 3: Depth→MLSD (Чистая геометрия комнаты)

Ключевая инновация: **MLSD запускается на карте глубины, а не на оригинальной фотографии.**

Проблема: при запуске MLSD на обычном фото модель детектирует ложные линии от теней, паркетных узоров, плинтусов и текстурных переходов. Это искажает расчёт Vanishing Point.

Решение: Depth Anything V2 генерирует карту глубины, которая:
- ✅ Убирает все текстуры, тени и отражения
- ✅ Оставляет только чистую 3D-геометрию (стыки стен, потолок, пол, проёмы)
- ✅ Создаёт идеальный вход для MLSD — только структурные линии "коробки" комнаты

```
Оригинал фото → Depth Anything V2 → Depth Map → MLSD → Структурные линии → VP + Гистограмма углов + Масштаб
```

#### Этап 4: Виртуальная примерка (Collision Detection)
На этапе создания базы скрипт анализирует не только пустую комнату (`before`), но и готовый дизайн-проект (`_after`). Он вырезает "Отпечаток мебели" (силуэты шкафов, диванов, кроватей). 
При поиске скрипт виртуально накладывает эту мебель на вашу пустую комнату. Если чужой шкаф перекрывает ваше окно — вариант жестко штрафуется.

## 🚀 Как запустить

### Требования
- Python 3.11+
- PyTorch с поддержкой MPS (Apple Silicon) или CUDA
- Установите зависимости:
```bash
pip install torch transformers opencv-python numpy pillow fastapi uvicorn tqdm
```

> **⚠️ Apple Silicon (MPS):** Запускайте с переменной окружения:
> ```bash
> export PYTORCH_ENABLE_MPS_FALLBACK=1
> ```

### 1. Индексация базы
Положите фотографии пустых комнат и их готовые дизайны (с припиской `_after` в названии) в папку `database`.
```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python image_retrieval.py index database
```

### 2. Поиск похожей комнаты (CLI)
```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python image_retrieval.py search путь/к/вашей_картинке.jpg --top-k 5
```

### 3. Запуск FastAPI Микросервиса (API)
```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 uvicorn retrieval_api.main:app --port 8000
```

Сервер поднимется на `http://localhost:8000`.

### API Endpoints

#### `POST /search`
Поиск похожих комнат. Принимает файл картинки и возвращает ранжированный список.

**Параметры:**
- `image` (file) — фотография пустой комнаты
- `top_k` (int, default=5) — количество результатов

**Пример запроса:**
```bash
curl -X POST "http://localhost:8000/search?top_k=3" \
  -H "Content-Type: multipart/form-data" \
  -F "image=@test_room.jpg"
```

**Ответ (JSON):**
```json
{
  "results": [
    {
      "rank": 1,
      "before_filename": "image_19.jpg",
      "after_filename": "image_19_after.jpg",
      "after_image_base64": "...",
      "score": 2.556,
      "floor_iou": 0.86,
      "ceiling_iou": 0.0,
      "depth_similarity": 0.76,
      "vp_similarity": 0.92,
      "scale_similarity": 0.90,
      "collision_percent": 0.28,
      "collision_warning": true
    }
  ],
  "room_type": "Corner",
  "dominant_wall": "Left"
}
```

#### `POST /segment-doors`
Сегментация дверей на изображении. Возвращает маску дверных проёмов.
